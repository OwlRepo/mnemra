### F4 — Chat page "Retry last answer" button always renders

`apps/web/app/workspaces/[id]/chat/page.tsx` around line 600-602:
```tsx
<div className="mt-4">
  <Button variant="ghost" size="sm" onClick={() => void reload()}>
    Retry last answer
  </Button>
</div>
```
Renders unconditionally in the Sources panel, even when `messages.length === 0` (nothing to retry) or when the most recent message is from the user (no assistant answer to retry). Gate it: only render when there's at least one assistant message to retry — the simplest correct condition is checking whether `messages` contains any `role === 'assistant'` entry (matches the same "was there ever an answer" check the empty-state logic elsewhere in this file already relies on implicitly).

```tsx
{messages.some((message) => message.role === 'assistant') ? (
  <div className="mt-4">
    <Button variant="ghost" size="sm" onClick={() => void reload()}>
      Retry last answer
    </Button>
  </div>
) : null}
```
