### StorageService — `apps/api/src/storage/` (NEW)

#### `storage.service.ts`
- New dep: `@aws-sdk/client-s3`. Inject `ConfigService`. Implement `OnModuleInit`.
- Construct `S3Client({ endpoint: S3_ENDPOINT, region: S3_REGION, credentials: { accessKeyId: S3_ACCESS_KEY, secretAccessKey: S3_SECRET_KEY }, forcePathStyle: S3_FORCE_PATH_STYLE === 'true' })`.
- `onModuleInit()` → `ensureBucket()`: `HeadBucketCommand`; on 404/NotFound, `CreateBucketCommand`. Swallow "already exists". Do NOT crash app boot if storage is briefly unreachable — log a clear error (but missing config should fail loudly).
- `save(key: string, body: Buffer, contentType?: string): Promise<string>` → `PutObjectCommand({ Bucket, Key: key, Body: body, ContentType })`; return `key`.
- `getToTempFile(key: string): Promise<string>` → `GetObjectCommand`; stream `Body` to a temp file at `path.join(os.tmpdir(), \`mnemra-\${randomUUID()}-\${basename(key)}\`)`; return the temp path. (Keeps `extname` so Slice 3B's `loadDocument(tempPath)` picks the right loader.)
- `delete(key: string): Promise<void>` → `DeleteObjectCommand`.

#### `storage.module.ts`
- `@Module({ providers: [StorageService], exports: [StorageService] })`. (No controller.) Will be imported by Documents + Ingest modules in Slice 3B.

- Add nullable `documents.storage_key` (text).
- Change `chunks.document_id` FK to `ON DELETE CASCADE`.
- New migration `packages/db/drizzle/0001_*.sql` via `db:generate`.

Migration Danger Gate:
- Migration required? Yes (`0001`).
- Backfill? No (new column nullable; existing rows unaffected).
- Default/nullability? `storage_key` nullable, no default.
- Index/constraint impact? Drops + recreates `chunks` FK with cascade. `chunks` is empty → zero data impact.
- Existing data impact? None.
- Rollback? Drop the column; recreate the FK without cascade. Safe on dev (no prod data).
- Deployment ordering? Apply migration before Slice 3B deploy.
